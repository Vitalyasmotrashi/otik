

\section{Декодирование: \finalcode, \msglen{} символов}

% При декодировании рабочих отрезка два:
% 
% \begin{itemize}
% 
% \item отрезок, соответствующий раскодированной части сообщения ($\textcolor{red}{[l_{c}, t_{c})}$);
% 
% \item отрезок, соответствующий прочитанной части кода ($\textcolor{decodegreen}{[l_{b}, t_{b})} \subseteq \textcolor{red}{[l_{c}, t_{c})}$).
% 
% \end{itemize}
% 
% 
% При чтении очередного бита кода отрезок~$\textcolor{decodegreen}{[l_b,t_b)}$ делится пополам.
% 
% Если $\textcolor{decodegreen}{[l_b, t_b)}$ целиком помещается в~подотрезок  $\textcolor{red}{[l_c, t_c)}$, соответствующий какому-либо символу, $\textcolor{red}{[l_c, t_c)}$ заменяется на этот подотрезок, а~символ присоединяется к~раскодированной части сообщения.
% 
% Масштабирование выполняется, только когда оба отрезка (или, что то же самое, б\'{о}льший $\textcolor{red}{[l_c, t_c)}$) помещаются в~одной из половин~$[0, N-1]$.
% \bigskip


Входной поток бесконечен: $\finalcode{}00000...$
\bigskip

\renewcommand{\intscale}[3][\intshortlabel]{
  \tikzmath{
    real \hscalestart, \binsubdivlen;

    {

    }

    };

  \binlines[\binsubdivlen]{#3}

  \inthpoints
  \drawalphsubdiv
  
}

\renewcommand{\halfscalevalues}[2][]{
  \tikzmath{
    real \hscalestart;
    \hscalestart = #2;    
    %
    \lbin = (\lbin - \hscalestart * \xtickfinal) * 2;

    %
    \lsym = (\lsym - \hscalestart * \xtickfinal) * 2;
    \hsym = (\hsym - \hscalestart * \xtickfinal) * 2 + 1;
  } 
}
