Попытка унификации обозначений (пока везде разные)

X "--- источник сообщений, случайная величина
в сжатии "--- модель источника

(переименовать!) X "--- входная последовательность (она же сообщение) ==> переименовать! 
==> W? C? пока C

x "--- сообщение из источника, значение случайной величины
x (простые коды чисел) "--- число
x (floats) "--- любой бит
x, арифметический кодер "--- код сообщения C как число ==> в перспективе переименовать (в z?) но пока нет


I(X) "--- среднее количество информации в источнике (H "--- энтропия)

I(x) "--- количество информации в сообщении источника

code(x)  "--- код сообщения

A "--- алфавит
A_1 "--- первичный алфавит (до преобразования);
A_2 "--- вторичный 

a "--- символ алфавита (байты!)
a_i "--- символ алфавита с кодом i (a_i \neq a_j при i \neq j)

C "--- сообщение

c_i "--- символ сообщения на позиции i (c_i может быть равен c_j, и они равны какому-то a_k из алфавита)

α "--- какая-то цепочка байт (зависит от контекста?)


N, M "--- зависит от контекста
N для X "--- количество возможных сообщений
N и M для кодирования блоками "--- размер блока до и после
N для чисел "--- разрядность
N, арифметический кодер "--- длина рабочей области

n, m, k "--- зависит от контекста
n "--- длина исходного файла в символах (байтах?)
m "--- длина результирующих данных в байтах
k "--- длина исходного файла в блоках

bit[i], b_i бит на позиции i

S, Хаффман "--- объединённый узел

арифметический кодер:
l, t, λ, τ  "--- отрезки
Δ
β
D
B "--- выходной поток в битах
